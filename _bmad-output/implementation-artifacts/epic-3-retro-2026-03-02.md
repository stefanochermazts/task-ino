## Deep Story Analysis – Patterns Extracted

### What went well

1. **Local-first isolation held under stress**
   - Planning capture/mutation flows remained available even with sync disabled, failing, or partially configured.

2. **Guardrail-first architecture scaled**
   - Sync merge/reconcile paths reused invariant enforcement instead of introducing bypass write paths.

3. **Adversarial reviews improved correctness materially**
   - Multiple high/critical issues were detected and fixed before closure (merge ownership/temporal checks, plaintext E2EE bypass, reset/delete consistency).

4. **User-safe error messaging became consistent**
   - Raw technical exceptions were progressively removed from user-facing copy in sync, reset, and delete flows.

### What did not go well

1. **Implementation/file-list drift in story artifacts**
   - Some files changed during fix cycles but were initially missing from story file lists.

2. **Mock-heavy tests initially masked build/runtime issues**
   - Missing exports and integration gaps passed unit tests until build/integration checks were run.

3. **Security boundaries needed explicit negative tests**
   - E2EE boundary regressions were possible until plaintext rejection and key-clear lifecycle tests were added.

---

## Key Insights

1. **Build checks are required for review closure**
   - Unit pass is not enough; export/import integrity must be verified via build in every story touching module boundaries.

2. **Trust controls require explicit "reject" paths**
   - For sync/E2EE/deletion, negative-path tests are as important as happy paths.

3. **Determinism is cross-cutting, not story-local**
   - Timestamp/device tie-break, invariant checks, and user-safe status copy must align across all trust stories, not per-feature only.

4. **Local continuity semantics must be asserted in integration tests**
   - Reset/delete stories need integration proof that local planning state is preserved or removed exactly by intent.
